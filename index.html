Data Explorer and SQL workbench inside Schema Weaver

Data Explorer is part of Schema Weaver. It helps you browse PostgreSQL databases, filter live rows, inspect column statistics, export data, and use Resona AI for advanced analysis without leaving the explorer workflow.

Core Features Database Support Resona AI Exports Schema Weaver Apps Docs

Explore live data quickly

Data Explorer uses a two-panel layout with a schema-aware sidebar on the left and a high-performance data grid on the right. You can search tables, open previews, sort columns, apply filters, refresh results, and copy values directly from the grid.

Inspect columns with smarter grid behavior

Column widths are measured with canvas-based sizing and 95th-percentile sampling so wide datasets stay readable. Clicking a header opens column statistics such as null percentage, distinct counts, top values, and min or max details.

Move from browsing to SQL and advanced AI analysis

Data Explorer is part of a larger Schema Weaver workflow. You can inspect raw PostgreSQL data here, export what you need, open deeper query workflows in the SQL Editor, and use Resona AI when you want help drafting queries or running more advanced analysis.

What you can do in Data Explorer

  • Browse PostgreSQL schemas and tables from a searchable sidebar.
  • Sort and filter rows server-side for large datasets.
  • Show or hide columns based on the current task.
  • Open column statistics to inspect quality and value distribution.
  • Copy individual cell values directly from the grid.
  • Export current tables or full schemas in multiple formats.
  • Use Resona AI for query help and advanced analysis inside the explorer workflow.
  • Save outputs into dashboard, workspace, and artifact history views.

Advanced analysis supported by Resona AI

Export PostgreSQL data in multiple formats

Data Explorer supports CSV, JSON, NDJSON, Excel, and SQL exports for current tables, plus full schema export. This is useful for reporting, handoff to notebooks, stakeholder sharing, and moving selected data into follow-up workflows.

Export support is one of the strongest product-specific topics on this page and should remain prominent in both search snippets and on-page content.

Read-only by default

Data Explorer is documented as read-only by default, which helps communicate safer production exploration. That supports trust for users evaluating the tool for operational PostgreSQL access.
